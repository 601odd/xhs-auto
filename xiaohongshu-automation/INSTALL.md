# 小红书自动化发帖系统 - 安装指南

## 系统要求

- Node.js 16+ 
- Chrome/Chromium 浏览器
- 2GB+ 内存
- 支持的操作系统：Windows 10+, macOS 10.14+, Linux Ubuntu 18.04+

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd xiaohongshu-automation
```

### 2. 安装依赖

```bash
# 安装服务端依赖
npm install

# 安装客户端依赖
cd client && npm install
cd ..
```

### 3. 配置环境变量

复制环境变量模板并配置：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置以下关键参数：

```env
# OpenAI API配置（必需）
OPENAI_API_KEY=your_openai_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-3.5-turbo

# 服务器配置
PORT=3000
NODE_ENV=development

# 验证码服务（可选，用于处理登录验证码）
CAPTCHA_TOKEN=your_2captcha_token_here
```

### 4. 创建必要目录

```bash
mkdir -p logs/screenshots data
```

### 5. 启动应用

开发环境：
```bash
npm run dev
```

生产环境：
```bash
npm run build
npm start
```

### 6. 访问应用

打开浏览器访问：`http://localhost:3000`

## 详细配置说明

### OpenAI API配置

1. **获取API Key**
   - 访问 [OpenAI官网](https://platform.openai.com/api-keys)
   - 注册账号并创建API Key
   - 将API Key填入 `.env` 文件的 `OPENAI_API_KEY` 字段

2. **支持的模型**
   - `gpt-3.5-turbo` (推荐，成本较低)
   - `gpt-4` (质量更高，成本较高)
   - 其他兼容OpenAI格式的模型

3. **替代方案**
   - 可以使用其他兼容OpenAI接口的服务
   - 修改 `OPENAI_BASE_URL` 指向对应的API地址

### 浏览器配置

系统会自动下载Chromium，如需使用系统已安装的Chrome：

1. 找到Chrome安装路径
2. 在 `server/services/puppeteerService.js` 中配置 `executablePath`

### 反检测配置

系统已集成多种反检测技术：

1. **puppeteer-extra-plugin-stealth** - 隐藏自动化特征
2. **随机用户行为模拟** - 随机延迟、鼠标轨迹
3. **设备指纹随机化** - User-Agent、分辨率等
4. **代理支持** - 可配置代理IP（需额外配置）

## 使用指南

### 1. 登录小红书

1. 访问系统首页
2. 点击"登录小红书"
3. 输入手机号和密码
4. 如有验证码，在弹出的浏览器窗口中手动完成
5. 登录成功后会保存cookies，下次可自动登录

### 2. 生成AI内容

1. 进入"AI内容生成"页面
2. 填写公司名称、职位、内推码等信息
3. 点击"生成内容"
4. 可选择不同模板或生成变体

### 3. 发布笔记

**一键发布（推荐）：**
1. 进入"一键发布"页面
2. 填写内推信息
3. 点击"一键发布"，系统会自动生成内容并发布

**手动发布：**
1. 先在"AI内容生成"中生成内容
2. 进入"发布笔记"页面
3. 编辑标题、正文、标签
4. 点击"发布"

### 4. 定时发布

1. 进入"系统设置"页面
2. 配置定时任务
3. 选择发布时间和频率
4. 启动定时任务

## 常见问题

### 登录相关

**Q: 登录失败怎么办？**
A: 
1. 检查手机号和密码是否正确
2. 确保网络连接正常
3. 查看logs目录下的错误日志
4. 尝试手动完成验证码

**Q: 提示需要验证码？**
A: 
1. 系统会自动弹出浏览器窗口
2. 手动完成滑动验证或点击验证
3. 验证完成后关闭窗口即可

### AI内容生成

**Q: 提示API调用失败？**
A: 
1. 检查OPENAI_API_KEY是否正确
2. 确认账号有足够余额
3. 检查网络是否能访问OpenAI API
4. 尝试更换API_BASE_URL

**Q: 生成的内容质量不好？**
A: 
1. 提供更详细的附加信息
2. 尝试不同的AI模型
3. 使用内容变体功能
4. 手动编辑生成的内容

### 发布相关

**Q: 发布失败？**
A: 
1. 确认已成功登录小红书
2. 检查网络连接
3. 查看错误日志了解具体原因
4. 避免频繁发布，注意间隔时间

**Q: 提示账号被限制？**
A: 
1. 停止自动化操作
2. 手动登录小红书检查账号状态
3. 适当降低发布频率
4. 确保内容质量和原创性

## 安全注意事项

1. **账号安全**
   - 不要泄露登录密码
   - 定期更换密码
   - 避免在多个设备同时登录

2. **内容合规**
   - 确保发布内容符合平台规范
   - 避免发布重复或低质量内容
   - 不要进行垃圾信息传播

3. **使用频率**
   - 控制发布频率，避免被检测
   - 建议每天发布不超过3-5条
   - 设置随机的发布时间间隔

4. **风险提示**
   - 自动化操作存在账号被封禁的风险
   - 建议先用测试账号验证
   - 重要账号请谨慎使用

## 故障排除

### 日志查看

系统日志位于 `logs/` 目录：
- `combined.log` - 所有日志
- `error.log` - 错误日志
- `screenshots/` - 错误截图

### 常用命令

```bash
# 查看实时日志
tail -f logs/combined.log

# 重启服务
npm restart

# 清除缓存
rm -rf data/cookies.json
rm -rf logs/*

# 检查端口占用
lsof -i :3000
```

   - 配置代理服务器（如需要）
   - 调整超时时间设置
