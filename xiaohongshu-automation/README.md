# 小红书自动化发帖系统

## 项目简介

本项目实现小红书自动化发帖功能，支持每日发送内推信息，具备AI内容生成和反检测机制。

## 功能特性

1. **绕过平台检测机制**
   - 使用puppeteer-extra-plugin-stealth隐藏自动化特征
   - 模拟真实用户行为（随机延迟、鼠标轨迹）
   - 指纹浏览器技术避免设备指纹检测

2. **AI内容生成**
   - 集成OpenAI GPT或其他LLM模型
   - 支持自定义prompt模板
   - 内容去重和多样化生成

3. **技术架构**
   - 后端：Node.js + Express
   - 前端：Vue3 + Element Plus
   - 自动化：Puppeteer + Stealth插件
   - AI：支持多种LLM接口

## 核心技术方案

### 1. 反检测技术

基于搜索结果，主要采用以下技术：

- **puppeteer-extra-plugin-stealth**: 隐藏Chrome headless特征
- **设备指纹随机化**: 随机生成User-Agent、屏幕分辨率等
- **行为模拟**: 模拟真实用户的点击、输入、滚动行为
- **时间随机化**: 添加随机延迟，避免规律性操作

### 2. 内容生成策略

- **模板化prompt**: 为内推信息设计专门的提示词模板
- **内容变化**: 每次生成不同表达方式，避免重复检测
- **关键词轮换**: 内推码和关键信息的多样化表达

### 3. 发布策略

- **定时发布**: 设置合理的发布频率和时间
- **内容质量**: 确保生成内容符合平台规范
- **监控机制**: 实时监控账号状态和发布结果

## 参考资料

基于调研，主要参考了以下开源项目和技术方案：

1. **xhs-toolkit**: 基于MCP协议的小红书自动化工具包
2. **xhs-mcp-server**: 支持图文和视频笔记自动发布
3. **小飞薯RPA**: 提供反检测和自动化运营功能

## 风险提示

1. **平台风险**: 小红书对自动化操作有严格检测，存在账号封禁风险
2. **合规性**: 必须遵守平台用户协议和社区规范
3. **技术更新**: 平台反作弊策略可能更新，需持续维护
